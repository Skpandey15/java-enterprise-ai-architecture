# Contributing

Thank you for helping improve this open architecture handbook.

## Good contributions

- Correct a technical inaccuracy with supporting references
- Add a clearly explained trade-off or failure scenario
- Improve capacity assumptions, security, observability, or cost analysis
- Add an original diagram or runnable reference implementation
- Propose an Architecture Decision Record (ADR)

## Contribution process

1. Open an issue describing the problem or proposed topic.
2. Fork the repository and create a focused branch.
3. Follow the relevant documentation template.
4. Explain assumptions and avoid absolute technology claims.
5. Open a pull request linked to the issue.

## Content standards

- Write original material and cite external sources.
- Do not copy articles, books, courses, or diagrams without permission.
- Separate facts from opinions and context-dependent recommendations.
- Use precise terminology.
- Include failure behaviour and operational concerns.
- Never include credentials, tokens, personal data, or proprietary information.

## Review checklist

- Are requirements and assumptions explicit?
- Are important alternatives and trade-offs documented?
- Are consistency, security, observability, resilience, and cost addressed?
- Are diagrams consistent with the written flows?
- Are links valid and sources properly attributed?
